# Security Policy

Do not open public issues for vulnerabilities, secrets, private data exposure, or exploitable implementation details.

Report security concerns privately to the organisation maintainers.

Never commit:

- API keys
- access tokens
- customer data
- private emails
- payment exports
- production secrets
- personal information
- private client or partner strategy notes

Public Kastel repositories should contain specifications, templates, examples, and sanitised reference material only.
