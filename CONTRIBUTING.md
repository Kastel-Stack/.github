# Contributing to Kastel Stack

Kastel Stack contributions should preserve the public/private boundary, source-of-truth discipline, human supervision, and anti-capture posture of the project.

Good first contributions include documentation fixes, schema improvements, examples, policy templates, connector notes, and review-flow clarifications.

Do not include private customer data, production secrets, private business operating details, payment exports, or partnership-sensitive material in public repositories.

## Contribution Flow

1. Open an issue or discussion for non-trivial design changes.
2. Keep pull requests focused on one concept or repository area.
3. Explain the source-of-truth implication of any schema or connector change.
4. Add examples when changing public templates or intent structures.
5. Flag any privacy, claims, payments, security, or governance implications.
